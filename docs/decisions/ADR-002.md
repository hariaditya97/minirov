# ADR-002: Plain Python middleware for v1 over ROS 2

## Status
Accepted

## Context
The miniROV requires a software layer between ArduSub and the LLM — handling MAVLink communication, sensor aggregation, camera streaming, and command execution. ROS 2 is the standard middleware for marine robotics. However, ROS 2 introduces significant setup complexity, a steep learning curve, and additional failure modes for a first build.

## Decision
Implement the v1 middleware as a plain Python application (minirov-brain) with direct pymavlink integration. ROS 2 refactoring is planned for v2 once the system is proven in water.

## Consequences
- Faster initial development — no ROS 2 node boilerplate
- Simpler debugging — standard Python tooling throughout
- Lower barrier to contribution from Python developers
- Direct transferability of architecture to ROS 2 in v2 — each Python module maps cleanly to a ROS 2 node
- No built-in message passing, service discovery, or transform tree in v1
