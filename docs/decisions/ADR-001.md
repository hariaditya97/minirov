# ADR-001: Offline-first LLM inference

## Status
Accepted

## Context
The miniROV is deployed in UK coastal and freshwater environments where internet connectivity cannot be guaranteed. A cloud-based LLM API introduces a hard dependency on network availability — if connectivity drops mid-dive, the autonomous control layer fails entirely. Additionally, operator commands describe real underwater locations and mission objectives, raising data privacy concerns if transmitted to third-party servers.

## Decision
Use Ollama running locally on the M4 Pro MacBook for all LLM inference. Models (qwen2.5:14b for command interpretation, llama3.2-vision:11b for visual analysis) are downloaded once and run entirely offline thereafter.

## Consequences
- No internet dependency during field operations
- Sub-second local inference latency versus 1-3 second cloud round trips
- Complete data privacy — no operator commands or camera frames leave the device
- Limited to models that fit within 18GB unified memory on the M4 Pro
- No access to frontier model capability (GPT-4, Claude) during operations
- Model updates require manual pulls rather than automatic API upgrades
