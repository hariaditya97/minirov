# ADR-004: Supervised autonomy over full autonomy

## Status
Accepted

## Context
The LLM integration could be designed for two operational modes: supervised autonomy (operator confirms all non-trivial actions before execution) or full autonomy (LLM executes commands without operator confirmation). Full autonomy would enable faster mission execution and reduce operator workload.

## Decision
Implement supervised autonomy for v1. Every non-trivial LLM-generated action is presented to the operator with reasoning and safety notes before execution. The operator confirms with y/n. Emergency surface command executes immediately without confirmation.

## Consequences
- Safety guaranteed by human in the loop — LLM cannot execute unsafe actions without operator awareness
- Hallucination risk mitigated — operator sees LLM reasoning before any physical action
- Operator builds trust in system behaviour gradually before considering full autonomy
- Slower mission execution than full autonomy — confirmation adds 5-30 seconds per action
- Operator fatigue on long missions — high confirmation frequency becomes burdensome
- Clear upgrade path to full autonomy in v2 once system behaviour is characterised and trusted
