# ADR-007: Operator Interface and Onboard Inference for Field Deployment

## Status
Proposed

## Context
Current deployment requires a MacBook at the surface for two functions:
1. Ollama LLM inference (text commands + vision analysis)
2. Operator interface (terminal UI for command input and state monitoring)

This creates a dependency on a laptop in potentially wet, sandy field conditions.
A simpler, more robust field setup is desirable for solo beach and lake deployment.

## Decision
Replace MacBook dependency with two changes:

**1 — Onboard LLM inference on Raspberry Pi 5**
Run a quantised multimodal model (llama3.2-vision 3B, 4-bit quantised) directly
on the Pi 5 using llama.cpp or Ollama's ARM build. Handles both natural language
command interpretation and vision frame analysis. Inference will be slower
(3-8 tokens/second versus ~40 on M4 Pro) but fully functional for supervised
autonomy where operator confirmation introduces natural latency anyway.

**2 — Web operator interface**
Replace terminal operator_node UI with a lightweight web server running on the Pi.
Serves a single-page interface showing:
- Live vehicle state (depth, battery, mode, attitude)
- LLM observations and warnings
- Command input box
- LLM response with confirm/cancel buttons
- Mission log of executed commands

Operator connects iPhone or any device to Pi's WiFi hotspot and opens Safari.
No app development required. No App Store. Works on any device.
QGroundControl iOS handles manual override and parameter editing independently.

## Consequences
**Positive:**
- No MacBook required in the field
- Simpler, more robust field setup
- iPhone becomes the only surface device needed
- Eliminates tether ethernet bandwidth requirement for LLM inference
- Pi fully self-contained — vehicle operates independently of surface computer

**Negative:**
- Inference significantly slower on Pi 5 vs M4 Pro
- Pi 5 thermal management under sustained inference load needs monitoring
- Web interface development adds scope to Phase 8
- Quantised model may have lower accuracy than full precision model on Mac

## Alternatives Considered
- Custom iOS app — rejected, significant development overhead, App Store process
- Jetson Orin Nano as companion computer — rejected for v1, revisit for v2
  if Pi 5 inference speed proves inadequate
- Keep MacBook dependency — rejected, impractical for solo field deployment

## Implementation
Phase 8 objective. Prerequisite: Phase 7 field deployment verified working
with MacBook. Replace MacBook dependency only after system proven reliable.
